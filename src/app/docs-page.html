Lit Elements Documentation

lit-button

A customizable button component with primary and ghost variants.

Properties

  • label (string): The text displayed on the button. Default: "Button"
  • variant ('primary' | 'ghost'): The visual style variant. Default: "primary"
  • disabled (boolean): Whether the button is disabled. Default: false

Events

  • lit-click: Emitted when the button is clicked (only if not disabled)

Usage Example

<lit-button 
  label="Add" 
  variant="primary"
  (lit-click)="handleClick()"
/>

Behavior

  • Primary variant: Displays with default button styling (neumorphism dark theme)
  • Ghost variant: Displays with transparent/subtle styling
  • When disabled, the button cannot be clicked and appears visually disabled
  • Click events bubble and are composed for use across shadow DOM

lit-checkbox

A checkbox component with label support and checked state management.

Properties

  • label (string): The text displayed next to the checkbox
  • checked (boolean): Whether the checkbox is checked. Default: false

Events

  • checked-change: Emitted when the checkbox state changes. Detail contains the new checked state (boolean)

Usage Example

<lit-checkbox
  [attr.label]="task.text"
  [attr.checked]="task.done ? '' : null"
  (checked-change)="onToggle($event)"
/>

Behavior

  • Clicking the checkbox toggles its checked state
  • The label text updates styling when checked (strikethrough effect)
  • Events bubble and are composed for cross-shadow DOM communication
  • Uses native HTML checkbox input for accessibility

lit-textbox

A text input component with placeholder support and keyboard event handling.

Properties

  • value (string): The current text value. Default: ""
  • placeholder (string): Placeholder text when input is empty. Default: "Enter todo"

Events

  • value-change: Emitted when the input value changes. Detail contains the new value (string)
  • enter-press: Emitted when the Enter key is pressed

Usage Example

<lit-textbox
  [attr.value]="inputValue()"
  placeholder="Add a new task"
  (value-change)="onInputChange($event)"
  (enter-press)="submitForm()"
/>

Behavior

  • Emits value-change event on every input change
  • Pressing Enter triggers the enter-press event (useful for form submission)
  • Events bubble and are composed for framework integration
  • Uses native HTML text input for accessibility

lit-navbar

A navigation bar component with tab switching functionality.

Properties

  • title (string): The brand/title text displayed on the navbar. Default: "Todo Manager"
  • active (string): The currently active tab ('personal' or 'professional'). Default: "personal"

Events

  • view-change: Emitted when a tab is clicked. Detail contains the selected view ('personal' | 'professional')

Usage Example

<lit-navbar
  title="Todo Workspace"
  [attr.active]="currentView()"
  (view-change)="onViewChange($event)"
/>

Behavior

  • Displays two tabs: "Personal" and "Professional"
  • Active tab is visually highlighted based on the active property
  • Clicking a tab emits view-change event with the selected view
  • Events bubble and are composed for routing integration
  • Styled with neumorphism dark theme for visual consistency

Design System

All Lit elements use a consistent design system with:

  • Dark Neumorphism Theme: Soft shadows and subtle depth effects
  • Color Palette: Carefully crafted dark colors with high contrast for accessibility
  • Spacing System: Consistent spacing scale (xs to huge) for visual rhythm
  • Typography: Hierarchical font sizes for clear content structure
  • Transitions: Smooth animations for interactive elements
